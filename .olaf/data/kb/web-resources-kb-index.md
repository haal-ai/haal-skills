## Usage Guidelines

### Fetch Priority
1. **Search local KB first** (`.olaf/data/kb/`, `.olaf/data/product/context/`)
2. **Check web resources index** (this file) for matching keywords
3. **Fetch official docs** if keywords match
4. **Use semantic search** to find related resources not in index

### Intelligent Fetching
- **Keyword Detection**: Match user question keywords to resource keywords
- **Context Awareness**: Understand which technology is being asked about
- **Multi-Fetch Strategy**: Fetch primary resource + related resources if needed
- **Fallback Search**: If not in index, infer official docs URL pattern

### URL Pattern Inference
Common patterns for official documentation:
- `https://docs.{technology}.com/` or `https://{technology}.io/docs/`
- `https://www.{technology}.org/documentation/`
- `https://developer.{technology}.com/docs/`
- `https://{technology}.readthedocs.io/`
