# Generate Post From Plan - Skill Description

## Overview

The **generate-post-from-plan** skill transforms presentation content or technical documentation into engaging blog posts tailored for development teams and IT professionals. It offers two distinct writing styles - brochure (structured) and conversational (narrative) - to match different communication needs.

## Purpose

This skill addresses the common challenge of converting technical presentations and documentation into accessible, engaging blog content that resonates with development teams, IT managers, and technical professionals.

## Key Capabilities

- **Multi-Style Content Generation**: Choose between brochure (structured) or conversational (narrative) styles, or generate both
- **Source Material Flexibility**: Process presentation plans, technical documentation, or project information
- **Professional Quality Standards**: Ensures factual accuracy, appropriate tone, and optimal reading length
- **Template-Driven Consistency**: Uses proven templates for reliable, professional output
- **Multi-Language Support**: Generates content in English, French, Spanish, or German

## Use Cases

### Ideal Scenarios
- Converting presentation plans into blog posts for team communication
- Transforming technical documentation into accessible content
- Creating multiple content versions (structured vs. narrative) from single source
- Generating team-focused content for IT announcements or updates

### Sample Input/Output
**Input:** Presentation plan about new development framework
**Output:** Professional blog post(s) with compelling title, structured content, and clear call-to-action, targeting under 2-minute reading time

## Technical Specifications

- **Input Parameters**: Source content, style preference, target audience, language, reading time target
- **Output Format**: Markdown files with professional blog post structure
- **Word Count Target**: 300-600 words for under 2-minute reading
- **Templates**: Dedicated templates for brochure and conversational styles

## Integration

This skill works well in content creation workflows, particularly when combined with:
- Presentation planning skills
- Technical documentation generation
- Multi-format content creation workflows
- Team communication strategies

## Quality Standards

- Maintains factual accuracy from source material
- Ensures professional tone appropriate for IT audience
- Uses clear, concise language (max 20 words per sentence)
- Includes actionable next steps
- Avoids fictional testimonials or invented statistics

## Success Metrics

- Generated content stays within 300-600 word target
- Reading time under 2 minutes
- Professional tone maintained throughout
- Clear value proposition communicated
- Actionable call-to-action included
