Analyze API Consistency - Tutorial¶

  1. From your workspace root (directory containing .olaf/), identify the target project path.
  2. Run full code-mapper foundation analysis:

bash cd <workspace-root> python scripts/code-mapper/run.py --foundation <project-path>

  1. Invoke the analyze-api-consistency skill and provide:
  2. project_path (absolute or workspace-rootâ€“relative).
  3. code_mapper_path if different from the default scripts/code-mapper.
  4. Optional custom output directory under .olaf/work/staging/code-mapper/.

  5. One or more target modules/packages to analyze.

  6. The skill will:

  7. Reuse or rerun full foundation as needed.
  8. Extract public API signatures for the selected modules from the code-map.
  9. Analyze naming, parameter, and return-pattern consistency.

  10. Generate a markdown report following templates/api-consistency-analysis-structure.md.

  11. Review the report and, if desired, confirm a path under .olaf/work/staging/code-mapper/<repo-name>/ to save it (for example api-consistency-analysis.md).
