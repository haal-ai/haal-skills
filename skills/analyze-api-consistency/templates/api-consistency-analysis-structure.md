# API Consistency Analysis - Report Structure

## 1. Extracted Public Function Signatures
- Per module/package, list all public functions/methods with full signatures.

## 2. Consistency Issues Identified
- Group issues by theme (e.g., repository reference pattern, naming, parameters, returns).
- For each issue, include a short description and affected functions.

## 3. Severity Summary
- Table summarizing issues by severity (HIGH/MEDIUM/LOW).

## 4. Canonical Patterns (Recommendations)
- Recommended patterns for repository references, verbs, parameter names, etc.

## 5. Detailed Impact Analysis for High Impact Issues
- For each HIGH impact issue, describe current state, proposed unified API, and migration impact.

## 6. Consistency Improvement Roadmap
- Immediate / Next / Future phases with concrete tasks.

## 7. Risk Assessment
- Overall risk level, mitigating factors, potential risks, and rollback plan.

## 8. Summary
- Current state, key recommended actions, and expected benefits.

## 9. References
- Pointers to foundation outputs (e.g., CODE_MAP file) and any related analyses/templates.
