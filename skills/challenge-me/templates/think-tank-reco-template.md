## Placeholder Guide

- `{subject}`: The topic or subject area for ideation
- `{session_identifier}`: Session ID in format `<subject-3-words>-YYYYMMDD-HHMM`
- `{timestamp}`: Current timestamp in YYYYMMDD-HHmm format
- `{cycle_count}`: Number of cycles completed
- `{go|no-go|conditional}`: The primary recommendation type
- `{GO|NO-GO|CONDITIONAL}`: Uppercase version for emphasis
- `{high|medium|low}`: Rating levels for various assessments
- `{list_*}`: Bulleted or numbered lists
- `{describe_*}`: Descriptive paragraphs
- `{summarize_*}`: Summary paragraphs
- `{if_*}`: Conditional content based on recommendation type
- `{repeat_for_*}`: Repeat the block for additional items
- `{action_N_with_owner_and_deadline}`: Specific action with responsible party and due date
- `{person_hours_or_person_days}`: Effort estimate
- `{cost_estimate}`: Budget estimate with currency
- `{duration_estimate}`: Time estimate for completion

## Notes

- Recommendations must be honest, even if negative
- If recommending NO-GO, provide clear alternatives to study
- All recommendations must be backed by evidence from cycles
- Risk assessment should be comprehensive and realistic
- Next steps must be specific, actionable, and time-bound
- Resource requirements should be detailed and realistic
- Success criteria must be measurable and achievable
- Include evidence from all sources: codebase, documentation, web, user responses
- Unresolved questions should trigger recommendations for further study
