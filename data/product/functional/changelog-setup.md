# Changelog Setup

## Purpose
Established dual changelog system for OLAF project tracking.

## Structure
- **Functional Changelog**: Business features, user stories, functional improvements
- **Technical Changelog**: Code changes, architecture updates, technical debt

## Format
- Daily entries using YYYY-MM-DD format
- Entries: `- Type: Description [Details: subject.md]`
- Detail files stored in respective functional/technical subfolders
