# Tell Me

## Overview

Lightweight knowledge retrieval skill that searches existing documentation artifacts before suggesting analysis. Returns fast, accurate answers from YOUR repository's documentation with source citations.

## Purpose

Provides instant access to project knowledge by searching local artifacts first, eliminating unnecessary analysis and providing immediate answers to common project questions.

## Usage

**Command**: `tell me` | `ask about` | `info about`

**Protocol**: Act

**When to Use**: When you need quick answers about your project from existing documentation.

## Parameters

### Required Inputs
- **Question or Topic**: What you want to know about the project

### Optional Inputs
- **Context**: Specific area or scope to search (e.g., "build process", "coding standards")
- **Detail Level**: Brief summary or comprehensive explanation

### Context Requirements
- `.olaf/data/product/{repo}/` folder with documentation (generated by `onboard-me`)
- OR `.olaf/data/practices/` folder with team practices

## Output

**Deliverables**:
- Synthesized answer from existing artifacts
- Source citations with file paths and line numbers
- OR suggestion to run appropriate analysis skill if no documentation exists

**Format**: Markdown with sources

## Search Locations (Priority Order)

1. `.olaf/data/product/{repo-name}/` - Project-specific documentation
2. `.olaf/data/practices/` - Team practices and standards
3. `README.md`, `CONTRIBUTING.md` - Root documentation
4. `.olaf/data/kb/` - Knowledge base entries

## Related Skills

- **onboard-me**: Generates project documentation that tell-me searches
- **analyze-contributor-risk**: Generates contributor analysis
- **measure-ai-impact**: Generates AI impact reports

## Tips & Best Practices

- Run `olaf onboard-me` first to generate comprehensive documentation
- Ask specific questions for better matches
- Use natural language - no need for exact keywords
- Check suggested skills if documentation doesn't exist
- Browse available documentation with `olaf tell me` (no question)

## Limitations

- Only searches existing artifacts, doesn't generate new analysis
- Requires `onboard-me` to have run first for best results
- Quality depends on available documentation
- Cannot answer questions beyond repository scope
- No access to external/generic knowledge bases

## Dependencies

- **onboard-me skill**: For generating initial documentation artifacts
- **Local documentation**: Requires `.olaf/data/product/` or `.olaf/data/practices/` folders

## How It Works

1. **Parse Question**: Extracts keywords and intent from user's question
2. **Search Artifacts**: Searches documentation files in priority order
3. **Rank Results**: Scores relevance of found content
4. **Synthesize Answer**: Combines relevant sections from top matches
5. **Cite Sources**: Provides file paths and line numbers for all sources
6. **Fallback**: Suggests appropriate analysis skill if no documentation exists
